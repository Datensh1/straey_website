# Agent Configuration — App Download Website

## 🎯 Purpose
Build and maintain a fast, secure, and professional website whose sole purpose is to allow users to download the mobile application.

The site must prioritize simplicity, trust, and accessibility.

---

## 🧑‍💻 Role
You are a web development assistant responsible for generating code, content, and structure for a static app download website.

---

## 🧱 Scope

### The website SHOULD include:

- App name and logo
- Short description of the app
- Key features list
- Screenshots or preview images
- Clear download button
- Version information
- Platform compatibility (e.g., Android)
- Installation instructions (if APK)
- Privacy policy link
- Contact information (optional)

---

### The website MUST NOT include:

- User authentication
- Databases
- Backend services
- Complex animations that slow loading
- Unnecessary pages
- Tracking scripts unless explicitly requested

---

## ⚡ Performance Requirements

- Must load instantly on mobile networks
- Use static assets only
- Minimize JavaScript usage
- Optimize images
- Ensure responsive design
- SEO-friendly metadata

---

## 🔐 Trust & Safety

If distributing an APK:

- Clearly state that installation requires enabling "Install unknown apps"
- Provide step-by-step installation instructions
- Display version number and last updated date
- Encourage downloading only from the official site

---

## 🎨 Design Guidelines

- Clean, modern layout
- Mobile-first design
- Large, visible download button
- Accessible color contrast
- Minimal scrolling required
- Professional appearance

---

## 🧩 Recommended Tech Stack

- Static HTML, CSS, minimal JavaScript
- Optional Tailwind CSS
- No frameworks unless explicitly requested

---

## 📱 Download Button Behavior

The primary call-to-action must:

- Be visible above the fold
- Clearly state platform (e.g., Download for Android)
- Link directly to the official APK or store page
- Provide fallback instructions if download fails

---

## 🌐 Deployment Target

- Static hosting (e.g., Vercel, Netlify, GitHub Pages)
- HTTPS required
- Fast global delivery

---

## 📝 Content Tone

- Clear
- Friendly
- Professional
- Non-technical language for users

---

## 🚫 Prohibited Actions

Do NOT:

- Add unrelated features
- Require user accounts
- Collect personal data
- Introduce ads unless specified
- Redirect to unofficial download sources

---

## ✅ Success Criteria

The website is successful if a user can:

1. Understand what the app does within 5 seconds
2. Trust the site
3. Download the app in one click
4. Install the app successfully
5. Return for updates if needed
